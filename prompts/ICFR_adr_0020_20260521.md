## 3. 구체 구현 지침

### 3.1 `backend/app/models/base.py` 수정

#### Before (현재 — 작업6 결과)

```python
from uuid import UUID, uuid4
from sqlalchemy.dialects.postgresql import UUID as PG_UUID
from sqlalchemy.orm import Mapped, mapped_column

class UUIDPrimaryKeyMixin:
    id: Mapped[UUID] = mapped_column(
        PG_UUID(as_uuid=True),
        primary_key=True,
        default=uuid4,
        nullable=False,
    )
```

#### After (변경 후)

```python
from uuid import UUID
from uuid_utils import uuid7 as _uuid7_native  # Python 3.13 미만 호환
from sqlalchemy.dialects.postgresql import UUID as PG_UUID
from sqlalchemy.orm import Mapped, mapped_column


def uuid7() -> UUID:
    """기본 PK 생성 — UUIDv7 (시간 기반 정렬, 인덱스 효율 ↑).
    
    uuid_utils 패키지가 반환하는 객체를 표준 UUID 로 변환.
    """
    return UUID(str(_uuid7_native()))


class UUIDPrimaryKeyMixin:
    """모든 모델의 PK 정의 — UUIDv7 자동 생성.
    
    ADR-0020 정책:
    - 기본: UUIDv7 (Python 애플리케이션 생성, 시간 기반)
    - 옵션: 호출 시 UUIDv4 또는 사용자 지정 UUID 사용 가능
    
    사용 예시 (특수 상황):
        # v4 가 필요한 경우 (예: 외부 시스템 호환):
        from uuid import uuid4
        instance = MyModel(id=uuid4(), ...)
        
        # 미리 정해진 ID 사용 (예: 마이그레이션):
        instance = MyModel(id=predetermined_id, ...)
    """
    id: Mapped[UUID] = mapped_column(
        PG_UUID(as_uuid=True),
        primary_key=True,
        default=uuid7,  # ← v7 기본
        nullable=False,
    )
```

> ⚠️ **변경 원칙 (사용자 룰 — 코드 가볍게)**:
> - **추상화 추가 금지** — UUID 생성 팩토리·전략 패턴 등 사용 안 함
> - **함수 1개로 끝** — `uuid7()` 만 추가, 다른 헬퍼·매니저 등 만들지 말 것
> - **import 최소화** — 필요한 것만 import

### 3.2 `backend/requirements.txt` 추가

```
uuid-utils>=0.7.0
```

> 💡 패키지 선택 근거: `uuid-utils` 는 Rust 기반 고성능 구현. UUIDv7 생성 속도 마이크로초 단위. 사용자 "성능 우선" 원칙 충족.

대안 (참고만, 본 작업에선 미사용):
- `uuid6` — 순수 Python, 약간 느림
- Python 3.13 표준 uuid 모듈 — 환경 의존성

### 3.4 검증 (재시드 후)

```sql
-- DB 접속
SELECT id, code FROM processes;
```

기대 결과: `id` 가 다음 패턴이어야 함 (UUIDv7 시간 기반 prefix):

```
id (UUIDv7 예시)
-------------------------------------
01902a1f-XXXX-7XXX-YYYY-ZZZZZZZZZZZZ
└─ 처음 48 bits 가 timestamp_ms ┘
└─ 13번째 글자가 "7" (버전 식별)
```

판별:
- 13번째 글자가 `7` → UUIDv7 ✅
- 13번째 글자가 `4` → UUIDv4 (이전, 실패)

> 💡 UUID 형식 (8-4-4-4-12 chars) 에서 13번째 글자가 버전 식별자.

## 6. 주의사항 — 사용자 원칙 강조

### 6.1 절대 추가하지 말 것 (사용자 결정 "코드 가벼움 극도 경계")

다음과 같은 추상화·구조 추가 **금지**:

- ❌ `UUIDFactory` 클래스
- ❌ `UUIDStrategy` 패턴
- ❌ `UUIDGenerator` 인터페이스
- ❌ `IDProvider` 추상 베이스 클래스
- ❌ `UUIDService` 서비스 레이어
- ❌ 설정 파일에 v7/v4 토글 (`UUID_VERSION=7` 같은 환경변수)
- ❌ Generic Type 으로 UUID 추상화

→ **추가해야 할 것은 `uuid7()` 함수 1개만**. 그 이상은 사용자 원칙 위반.

### 6.2 v4 호출 방식 (옵션)

사용자가 결정한 "v4 옵션 호출" 은 **별도 추상화 없이** 단순히:

```python
# 특수 상황 (예: 외부 시스템 호환)
from uuid import uuid4

instance = MyModel(id=uuid4(), ...)  # ← 호출자가 직접 지정
```

→ 이게 끝. Mixin 수정·팩토리 등 불필요.

### 6.3 성능 점검 (옵션)

작업 끝 후 마지막 검증으로 (선택):

```python
import time
from app.models.base import uuid7
from uuid import uuid4

t1 = time.perf_counter()
for _ in range(10000):
    uuid7()
t2 = time.perf_counter()

t3 = time.perf_counter()
for _ in range(10000):
    uuid4()
t4 = time.perf_counter()

print(f"uuid7: {(t2-t1)*1000:.2f}ms / 10000 calls")
print(f"uuid4: {(t4-t3)*1000:.2f}ms / 10000 calls")
```

→ 둘 다 마이크로초 단위. 성능 차이 무시 가능.
