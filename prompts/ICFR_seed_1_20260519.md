## 1. 작업 범위 (전체 그림)

### 1.1 본 작업이 만드는 것

5개 활성 모듈에 다음 4계층을 모두 만들어, **시드 데이터를 API 호출로 박을 수 있는 최소 시스템**을 완성:

```
[Layer 1] SQLAlchemy 모델 (DB 테이블)
[Layer 2] Pydantic 스키마 (Create/Read/Update)
[Layer 3] FastAPI 엔드포인트 (GET 리스트/POST 생성/GET 단건/PATCH 수정/DELETE 삭제)
[Layer 4] Alembic 마이그레이션 (DB 스키마 생성)

[+] backend/app/seeds/ 하위 모듈별 시드 파일이 위 API를 호출하여 데이터 생성
```

### 1.2 이전 작업과의 관계

- 작업3 완료 후 11개 모듈 라우터는 `/info` 엔드포인트만 보유
- 본 작업은 **5개 활성 모듈** (RCM, Remediation, Evidence, User Mgmt, Test) 에만 최소 CRUD 추가
- 나머지 6개 모듈 (Schedule, Scoping, EUC, IUC, Notification, Report) 은 Phase 1 이후
- bootstrap.py (작업2) 의 admin 자동 생성은 유지

### 2.2 RCM 모듈

```python
# models/rcm.py

class Process(Base, TimestampMixin):
    """업무 프로세스 (매출, 구매, 재고 등)"""
    __tablename__ = "processes"
    id: Mapped[int] = mapped_column(primary_key=True)
    code: Mapped[str] = mapped_column(String(20), unique=True, nullable=False)  # "P-SALES"
    name: Mapped[str] = mapped_column(String(100), nullable=False)
    description: Mapped[str | None] = mapped_column(Text, nullable=True)
    
    controls: Mapped[list["Control"]] = relationship(back_populates="process")

class RiskCategory(Base, TimestampMixin):
    """위험 분류 (E-Existence, C-Completeness 등 — Assertions)"""
    __tablename__ = "risk_categories"
    id: Mapped[int] = mapped_column(primary_key=True)
    code: Mapped[str] = mapped_column(String(10), unique=True, nullable=False)  # "E", "C", "V"
    name: Mapped[str] = mapped_column(String(50), nullable=False)
    description: Mapped[str | None] = mapped_column(Text, nullable=True)

class Control(Base, TimestampMixin):
    """통제 (RCM 핵심)"""
    __tablename__ = "controls"
    id: Mapped[int] = mapped_column(primary_key=True)
    code: Mapped[str] = mapped_column(String(20), unique=True, nullable=False)  # "C-001"
    name: Mapped[str] = mapped_column(String(200), nullable=False)
    description: Mapped[str | None] = mapped_column(Text, nullable=True)
    process_id: Mapped[int] = mapped_column(ForeignKey("processes.id"), nullable=False, index=True)
    frequency: Mapped[str] = mapped_column(String(20), nullable=False)  # "daily", "monthly", "quarterly"
    
    process: Mapped["Process"] = relationship(back_populates="controls")
    assertions: Mapped[list["ControlAssertion"]] = relationship(back_populates="control")

class ControlAssertion(Base, TimestampMixin):
    """통제와 위험 분류의 매핑 (N:M)"""
    __tablename__ = "control_assertions"
    id: Mapped[int] = mapped_column(primary_key=True)
    control_id: Mapped[int] = mapped_column(ForeignKey("controls.id"), nullable=False, index=True)
    risk_category_id: Mapped[int] = mapped_column(ForeignKey("risk_categories.id"), nullable=False, index=True)
    
    control: Mapped["Control"] = relationship(back_populates="assertions")
    risk_category: Mapped["RiskCategory"] = relationship()
```

### 2.3 Test 모듈

```python
# models/test_module.py

class TestRun(Base, TimestampMixin):
    """통제 평가 실행 (한 통제에 대한 1회 평가)"""
    __tablename__ = "test_runs"
    id: Mapped[int] = mapped_column(primary_key=True)
    control_id: Mapped[int] = mapped_column(ForeignKey("controls.id"), nullable=False, index=True)
    tester_id: Mapped[int] = mapped_column(ForeignKey("users.id"), nullable=False, index=True)
    test_date: Mapped[date] = mapped_column(Date, nullable=False)
    result: Mapped[str] = mapped_column(String(20), nullable=False)  # "pass", "fail", "n/a"
    notes: Mapped[str | None] = mapped_column(Text, nullable=True)
    
    steps: Mapped[list["TestStep"]] = relationship(back_populates="test_run", cascade="all, delete-orphan")

class TestStep(Base, TimestampMixin):
    """평가 절차의 한 스텝"""
    __tablename__ = "test_steps"
    id: Mapped[int] = mapped_column(primary_key=True)
    test_run_id: Mapped[int] = mapped_column(ForeignKey("test_runs.id"), nullable=False, index=True)
    step_order: Mapped[int] = mapped_column(Integer, nullable=False)
    description: Mapped[str] = mapped_column(Text, nullable=False)
    result: Mapped[str] = mapped_column(String(20), nullable=False)  # "pass", "fail"
    
    test_run: Mapped["TestRun"] = relationship(back_populates="steps")
```

### 2.5 Evidence 모듈

```python
# models/evidence.py

class EvidenceFile(Base, TimestampMixin):
    """증빙 파일 (메타데이터만; 파일은 MinIO 별도)"""
    __tablename__ = "evidence_files"
    id: Mapped[int] = mapped_column(primary_key=True)
    filename: Mapped[str] = mapped_column(String(255), nullable=False)
    mime_type: Mapped[str] = mapped_column(String(100), nullable=False)
    size_bytes: Mapped[int] = mapped_column(Integer, nullable=False)
    minio_key: Mapped[str | None] = mapped_column(String(500), nullable=True)  # Phase 1 활성화
    uploaded_by_id: Mapped[int] = mapped_column(ForeignKey("users.id"), nullable=False, index=True)
    
    links: Mapped[list["EvidenceLink"]] = relationship(back_populates="file", cascade="all, delete-orphan")

class EvidenceLink(Base, TimestampMixin):
    """증빙 파일과 다른 엔티티(통제·평가)의 연결"""
    __tablename__ = "evidence_links"
    id: Mapped[int] = mapped_column(primary_key=True)
    file_id: Mapped[int] = mapped_column(ForeignKey("evidence_files.id"), nullable=False, index=True)
    linked_entity_type: Mapped[str] = mapped_column(String(50), nullable=False)  # "control", "test_run", "deficiency"
    linked_entity_id: Mapped[int] = mapped_column(Integer, nullable=False, index=True)
    
    file: Mapped["EvidenceFile"] = relationship(back_populates="links")
```

> 💡 Phase 0 에서는 MinIO 업로드 로직은 생략. `minio_key` 만 nullable 컬럼으로 두고 Phase 1 에서 활성화. EvidenceFile API 도 메타데이터만 등록 가능하게.

class ProcessBase(BaseModel):

    name: str = Field(min_length=1, max_length=100)
    description: str | None = None

class ProcessCreate(ProcessBase):
    pass

class ProcessUpdate(BaseModel):
    name: str | None = Field(None, min_length=1, max_length=100)

class ProcessRead(ProcessBase):
    id: int
    created_at: datetime

    model_config = ConfigDict(from_attributes=True)

## 4. API 엔드포인트 (Layer 3)

### 4.1 표준 CRUD 패턴

각 모델에 대해 다음 5개 엔드포인트 (Process 예시):

```
GET    /api/rcm/processes              → 리스트 (페이지네이션)
POST   /api/rcm/processes              → 생성
GET    /api/rcm/processes/{id}         → 단건 조회
PATCH  /api/rcm/processes/{id}         → 부분 수정
DELETE /api/rcm/processes/{id}         → 삭제
```

### 4.2 모듈별 엔드포인트 트리

```
/api/rcm/
├── /info                       (작업3 기존)
├── /processes                  (CRUD)
├── /processes/{id}             (CRUD)
├── /risk-categories            (CRUD)
├── /risk-categories/{id}       (CRUD)
├── /controls                   (CRUD)
├── /controls/{id}              (CRUD)
└── /control-assertions         (CRUD)

/api/test/
├── /info
├── /runs                       (CRUD)
├── /runs/{id}
├── /runs/{id}/steps            (CRUD — nested under run)
└── /runs/{id}/steps/{step_id}

/api/remediation/
├── /info
├── /deficiencies               (CRUD)
├── /deficiencies/{id}
├── /plans                      (CRUD)
└── /plans/{id}

/api/evidence/
├── /info
├── /files                      (CRUD — 메타데이터만, MinIO 업로드는 Phase 1)
├── /files/{id}
├── /links                      (CRUD)
└── /links/{id}

/api/user-mgmt/
├── /info
├── /users                      (READ-only — 기존 users 활용)
├── /users/{id}
├── /roles                      (CRUD — UserRole)
└── /roles/{id}
```

### 4.3 인증

모든 엔드포인트에 `Depends(get_current_user)` 적용 (작업2 기존 패턴 활용).

### 4.4 페이지네이션

리스트 엔드포인트는 `?skip=0&limit=100` 쿼리 파라미터 지원. 응답:
```json
{
  "items": [...],
  "total": 50,
  "skip": 0,
  "limit": 100
}
```

### 4.5 응답 코드

- POST 성공: 201 Created
- DELETE 성공: 204 No Content
- 조회·수정: 200 OK
- 404 / 422 / 401 / 403 적절히

def test_create_and_get_process(authenticated_client):
    """프로세스 생성 + 조회 통합 테스트"""
    response = authenticated_client.post("/api/rcm/processes", json={

    assert response.status_code == 201
    process_id = response.json()["id"]
