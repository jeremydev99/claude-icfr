### 6.2 구현 지침 (사용자 원칙: 가볍게)

```python
# api/test_module.py 의 transition 엔드포인트

ALLOWED_TRANSITIONS = {
    "planned": {"in_progress"},
    "in_progress": {"completed"},
    "completed": {"approved"},
    "approved": set(),  # 변경 불가
}

@router.post("/runs/{run_id}/transition")
def transition_test_run(
    run_id: UUID,
    body: TransitionRequest,
    db: Session = Depends(get_db),
    user: User = Depends(get_current_user),
):
    """단일 함수로 처리. WorkflowEngine 같은 추상화 금지."""
    run = db.get(TestRun, run_id)
    if not run:
        raise HTTPException(404, "TestRun not found")
    
    current = run.status
    target = body.to_status
    
    # 허용 전이 검증
    if target not in ALLOWED_TRANSITIONS.get(current, set()):
        raise HTTPException(
            422, f"Cannot transition from '{current}' to '{target}'"
        )
    
    # 이력 기록 (TestStatusHistory)
    history = TestStatusHistory(
        test_run_id=run.id,
        from_status=current,
        to_status=target,
        changed_by_id=user.id,
        reason=body.reason,
    )
    db.add(history)
    
    # 상태 변경
    run.status = target
    
    # 특수 처리: approved 시 승인자·시점 기록
    if target == "approved":
        run.approved_by_id = user.id
        run.approved_at = datetime.now(timezone.utc)
    
    db.commit()
    db.refresh(run)
    return {"test_run": run, "history_entry": history}
```

> ⚠️ **절대 만들지 말 것**:
> - ❌ `WorkflowEngine` 클래스
> - ❌ `StateMachine` 라이브러리 도입
> - ❌ 전이 룰을 DB 에 저장 (`workflow_rules` 테이블 등)
>
> → ALLOWED_TRANSITIONS 는 **하드코딩**. 단순 dict.
